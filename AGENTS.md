# Antigravity 智能体全局规则 (AGENTS.md)

## 1. 核心运行环境与路径规范 (Environment & Paths)
- **纯正 Linux 假设**: 强制假设当前运行在纯粹的 Ubuntu Linux 环境中，忽略任何 Windows 特定的路径格式（如 `C:\` 或 `D:\`）。
- **高性能 I/O 路径**: 在处理海量训练数据集（例如视觉语言模型的图像与标注数据）或高频读写任务时，务必将工作区和数据目录放置在 WSL 原生文件系统内（如 `~/<project_name>` 或 `/home/$USER/`）。严禁在跨文件系统挂载点（如 `/mnt/c/` 或 `/mnt/d/`）下执行密集的 I/O 操作，以防引发极端的性能瓶颈。
- **权限管理**: 遇到 `Permission denied` 时，优先使用 `chmod`、`chown` 或 `ls -la` 检查和调整当前项目目录的所有权，避免在普通开发任务中滥用 `sudo` 提权。

## 2. 优先使用的终端命令 (Preferred Commands)
在自主排查报错、搜索文件或环境配置时，必须优先使用以下原生 Linux 工具链：
- **系统与算力监控**: 使用 `htop` 查看 CPU 与内存状态；在进行大模型算法实验或推理测试时，随时使用 `nvidia-smi` 或 `nvtop` 监控 GPU 显存占用，确保实验不会 OOM。
- **文本与日志分析**: 优先使用 `grep -rn`、`find . -name`、`awk` 和 `sed` 进行高效的代码检索和日志过滤。
- **网络与依赖**: 使用 `curl` 或 `wget` 下载测试数据和测试 API 连通性。
- **版本控制**: 在执行重大代码重构前（如调整核心算法逻辑），务必先执行 `git status` 检查当前状态，若有未保存的工作，需提醒我进行 commit 保存快照。

## 3. 开发栈与 AI 自动化规范 (Tech Stack & AI Tooling)
- **Python 隔离环境**: 所有的 Python 依赖安装 (`pip install`) 和脚本执行 (`python3`)，必须在激活的虚拟环境中进行。在执行任何 Python 相关的终端命令前，先检查并执行类似 `source .venv/bin/activate` 的操作。
- **Gemini CLI 协同**: 在使用 Gemini CLI 工具流进行代码生成、自动化排版调试或脚本执行时，请确保传递完整且正确的 Linux 绝对路径作为上下文输入。
- **智能体架构开发**: 在构建 AI Agent 架构（如基于 OpenClaw 等框架的集成与二次开发）时，严格遵循模块化设计，并在代码中预留详尽的 logging 接口，以便完整回溯智能体的感知与决策过程。

## 4. 代码生成与重构标准 (Output & Coding Style)
- **工业级标准**: 生成的代码需具备极高的健壮性，必须包含详尽的中文类型提示（Type Hints）和 Docstring 规范注释。
- **复杂逻辑注释**: 对于涉及多模态数据处理、多智能体协同控制或底层设备通信的代码，必须在核心逻辑块上方添加解释性注释，说明该算法设计的数学考量与参数边界。
- **无损修改**: 在跨文件修改代码时，严格保持原有的代码缩进风格，不要擅自删除与当前任务无关的已有功能代码。
